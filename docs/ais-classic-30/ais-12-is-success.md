# AIS經典論文 12：資訊系統成功模型

**English Title**: Information systems success: The quest for the dependent variable

**作者**: DeLone, W. H., & McLean, E. R.
**年份**: 1992
**期刊**: Information Systems Research
**卷期**: 3(1), 60–95
**DOI**: [10.1287/isre.3.1.60](https://doi.org/10.1287/isre.3.1.60)

## 📌 第一張：核心觀點卡

**本論文回顧 180+ 篇 IS 成功研究，提出整合性「IS 成功模型」，主張 IS 成功是多維度構念，包含系統品質、資訊品質、使用、使用者滿意度、個人影響、組織影響六個相互依賴的維度**

### 關鍵數據
- **文獻回顧**：180+ 篇 IS 成功實證研究（1981-1987）
- **引用次數**：30,000+ 次，IS 領域引用最多的論文之一
- **核心貢獻**：整合 IS 成功的多元測量，建立因果模型
- **六大維度**：系統品質、資訊品質、使用、使用者滿意度、個人影響、組織影響
- **影響力**：成為 IS 評估的標準框架

### D&M IS 成功模型（1992 原始版）

```
系統品質 (System Quality)  ──┐
                            ├──→ 使用 (Use) ────────┐
資訊品質 (Information Quality)─┤                    │
                            └──→ 使用者滿意度 ──────┤
                                (User Satisfaction)  │
                                    ↕                ↓
                                (相互影響)    個人影響 ──→ 組織影響
                                             (Individual) (Organizational)
```

### IS 成功的六大維度

| 維度 | 定義 | 測量範例 | IS 層次 |
|------|------|---------|---------|
| **系統品質** | 系統本身的技術特性 | 可靠性、回應時間、易用性 | 系統 |
| **資訊品質** | 系統產出資訊的品質 | 準確性、時效性、完整性 | 資訊 |
| **使用** | 系統被使用的程度和方式 | 使用頻率、使用時間、使用深度 | 使用 |
| **使用者滿意度** | 使用者對系統的整體滿意程度 | 整體滿意、重複使用意願 | 使用 |
| **個人影響** | 系統對個人的影響 | 決策品質、生產力、學習 | 影響 |
| **組織影響** | 系統對組織的影響 | 成本降低、營收增加、競爭力 | 影響 |

### 核心論點

**問題**：IS 成功該如何測量？過去研究使用各種不同指標，無法比較
**解決**：IS 成功不是單一構念，而是六個相互依賴的維度
**意涵**：評估 IS 成功需要多維度測量，且要理解維度間的因果關係

## ✍️ 第二張：Paraphrase 卡

這篇論文解決 IS 領域的根本問題：「什麼叫做成功的資訊系統？」過去研究各說各話——有人看使用率、有人看滿意度、有人看投資報酬率——DeLone & McLean 整合這些觀點，提出「IS 成功是多維度的，而且這些維度有因果關係」。

### 我的理解

關鍵洞察是**多維度 + 因果鏈**：

**過去混亂狀況**：
- 研究 A：使用率高 = 成功
- 研究 B：滿意度高 = 成功
- 研究 C：提升生產力 = 成功
- **問題**：無法比較、無法累積知識

**D&M 整合**：
- 這些都是「成功」，但處於**不同層次**
- 有**因果順序**：品質 → 使用/滿意度 → 影響
- 建立**整合框架**：所有研究可在同一框架下比較

**因果鏈邏輯**：

```
投入 (Input)
    ↓
系統開發 → 系統品質、資訊品質（過程品質）
    ↓
使用者接觸系統 → 使用、使用者滿意度（使用階段）
    ↓
產生效益 → 個人影響、組織影響（結果）
```

### 為何重要？

**實務意涵**：
- 不能只看使用率（可能強制使用但不滿意）
- 不能只看滿意度（可能滿意但沒效益）
- **需要全面評估**：品質 → 使用 → 影響

**研究意涵**：
- 提供共同語言（六大維度）
- 建立理論框架（因果關係）
- 促進知識累積（可比較、可整合）

### 對 IS 研究的影響

- **IS 評估標準**：成為評估 IS 的標準框架
- **理論整合**：整合 TAM、ECM、使用者滿意度等理論
- **實務應用**：企業評估 ERP、CRM 等系統的成功
- **後續更新**：DeLone & McLean (2003) 更新模型

## ❓ 第三張：問答卡

### Q1: 六大維度的因果關係為何？

**A**: DeLone & McLean 提出的因果路徑（基於文獻歸納）：

**主要因果鏈**：

**路徑 1：品質 → 使用**
- 系統品質 ↑ → 使用 ↑
- 資訊品質 ↑ → 使用 ↑
- 邏輯：好用、有用的系統才會被用

**路徑 2：品質 → 滿意度**
- 系統品質 ↑ → 使用者滿意度 ↑
- 資訊品質 ↑ → 使用者滿意度 ↑
- 邏輯：品質好讓人滿意

**路徑 3：使用 ↔ 滿意度（雙向）**
- 使用 → 滿意度（使用經驗影響滿意度）
- 滿意度 → 使用（滿意促進持續使用）
- 邏輯：正向循環

**路徑 4：使用 → 個人影響**
- 使用 ↑ → 個人影響 ↑
- 邏輯：用了才有影響

**路徑 5：滿意度 → 個人影響**
- 滿意度 ↑ → 個人影響 ↑（可能）
- 爭議：滿意但無效益？

**路徑 6：個人影響 → 組織影響**
- 個人影響 ↑ → 組織影響 ↑
- 邏輯：個人績效提升 → 組織績效提升

**完整因果圖**：

```
系統品質 ──┬──→ 使用 ──────┬──→ 個人影響 ──→ 組織影響
          │       ↕        │
資訊品質 ──┴──→ 滿意度 ────┘
              (雙向影響)
```

**醫療 AI 實例**：

**路徑追蹤**：
1. **系統品質**：AI 準確率高、回應快（高品質）
2. **資訊品質**：AI 建議相關、可信（高品質）
3. **使用**：醫師頻繁使用 AI（因為品質好）
4. **滿意度**：醫師對 AI 滿意（品質好 + 使用經驗好）
5. **個人影響**：診斷準確率提升、時間節省（使用產生效益）
6. **組織影響**：醫院醫療品質提升、成本降低（個人效益累積）

**失敗案例**（斷鏈）：
- 高使用 但 低滿意度（強制使用）→ 個人影響可能仍有，但長期不持續
- 高滿意度 但 低使用（非必要系統）→ 無個人/組織影響
- 高個人影響 但 低組織影響（個人效益未整合）

### Q2:「使用」和「滿意度」為何是雙向關係？

**A**: 這是 D&M 模型的關鍵創新，挑戰單向因果假設：

**傳統單向假設**：

**假設 A**：使用 → 滿意度
- 邏輯：用了才知道好不好
- 類似 ECM（期望確認理論）

**假設 B**：滿意度 → 使用
- 邏輯：滿意才會用
- 類似 TAM（態度 → 行為）

**D&M 雙向主張**：兩者**互相影響**，形成正向或負向循環

**正向循環**：
```
T1: 初次使用 → 正面經驗 → 滿意度 ↑
T2: 滿意度 ↑ → 持續使用 ↑
T3: 使用 ↑ → 更深入經驗 → 滿意度更 ↑
...（螺旋上升）
```

**負向循環**：
```
T1: 初次使用 → 負面經驗 → 滿意度 ↓
T2: 滿意度 ↓ → 減少使用 ↓
T3: 使用 ↓ → 技能下降 → 經驗更差 → 滿意度更 ↓
...（螺旋下降）
```

**實證支持**：

**支持「使用 → 滿意度」**：
- 使用經驗影響滿意度評價
- 首次使用者 vs 熟練使用者滿意度不同

**支持「滿意度 → 使用」**：
- 滿意的使用者更常使用系統
- Bhattacherjee (2001) ECM：滿意度預測持續使用

**調節變數**：**自願 vs 強制使用**

**自願使用情境**：
- 滿意度 → 使用（強烈影響）
- 不滿意 → 不用（可選擇）

**強制使用情境**：
- 滿意度 → 使用（影響減弱）
- 不滿意 → 仍得用（無選擇）
- 但：滿意度影響**使用深度**（表面用 vs 深度用）

**醫療 AI 應用**：

**正向循環範例**：
- 醫師初次使用 AI → 發現有用 → 滿意
- 滿意 → 更常使用 → 發現更多功能 → 更滿意
- 持續使用 → 習慣養成 → 成為工作流程一部分

**負向循環範例**：
- 醫師初次使用 AI → 建議不準 → 不滿
- 不滿 → 減少使用 → 未學會正確用法 → 效果更差
- 更不滿 → 完全放棄

**干預策略**：
- **初期**：確保正面首次經驗（避開複雜案例）
- **中期**：提供支援（防止負向循環）
- **長期**：持續改善（維持滿意度）

### Q3: 如何在實務中應用 D&M 模型評估 IS？

**A**: D&M 模型提供**評估框架**，但需要根據情境具體化測量：

**步驟 1：確定評估目標**

**情境 A**：評估新系統是否成功
- 焦點：全面評估六大維度
- 時機：系統上線 6-12 個月後

**情境 B**：診斷系統問題
- 焦點：識別薄弱環節（哪個維度最差？）
- 時機：使用率下降或滿意度低時

**情境 C**：比較系統方案
- 焦點：系統 A vs 系統 B 的六維度比較
- 時機：採購決策時

**步驟 2：選擇測量指標**

**系統品質**（技術面）：
- 客觀：系統可用率、回應時間、錯誤率
- 主觀：知覺易用性、系統可靠性（問卷）

**資訊品質**（內容面）：
- 客觀：資訊準確率、更新頻率
- 主觀：知覺資訊相關性、完整性（問卷）

**使用**（行為面）：
- 客觀：登入次數、使用時間、功能使用率（系統日誌）
- 主觀：自陳使用頻率、使用深度（問卷）

**使用者滿意度**（態度面）：
- 主觀：Bailey & Pearson 39 因素、Ives 13 因素簡化版

**個人影響**（效益面）：
- 客觀：決策品質、任務完成時間、錯誤率
- 主觀：知覺生產力、決策品質提升（問卷）

**組織影響**（整體效益）：
- 客觀：ROI、成本節省、營收增加
- 主觀：管理者評估組織效益

**步驟 3：資料蒐集與分析**

**資料來源**：
- 問卷：使用者滿意度、主觀評估
- 系統日誌：實際使用數據
- 訪談：深入了解影響
- 財務數據：組織層次效益

**分析方法**：
- **描述統計**：各維度平均分數
- **相關分析**：驗證因果關係（如品質 vs 滿意度）
- **結構方程模型**：驗證完整因果鏈
- **迴歸分析**：找出關鍵影響因素

**醫療 AI 評估範例**：

**情境**：評估 AI 診斷系統（上線 1 年）

**測量**：

| 維度 | 指標 | 資料來源 | 分數 |
|------|------|---------|------|
| 系統品質 | 可用率、回應時間 | 系統監控 | 4.2/5 |
| 資訊品質 | AI 準確率、相關性 | 驗證數據 + 問卷 | 3.8/5 |
| 使用 | 使用頻率、案例數 | 系統日誌 | 65% 醫師使用 |
| 滿意度 | 整體滿意度 | 問卷 | 3.5/5 |
| 個人影響 | 診斷時間、準確率 | 績效數據 | 時間 -15% |
| 組織影響 | 醫療品質、成本 | 醫院數據 | 待評估 |

**診斷**：
- **問題**：滿意度偏低（3.5/5）拖累使用率
- **根因**：資訊品質（AI 準確率僅 3.8/5）
- **建議**：改善 AI 模型 → 提升資訊品質 → 提升滿意度 → 提升使用

## 🤔 第四張：例外卡

### 疑點 1：過程模型 vs 因果模型的混淆

**D&M 主張**：模型呈現「過程」（Process Model）
**實務使用**：常被當成「因果模型」（Causal Model）

**混淆問題**：

**過程觀點**（D&M 原意）：
- 描述 IS 成功的**時間順序**
- 系統開發 → 使用 → 產生影響（時序）
- 不一定是嚴格因果

**因果觀點**（常見誤用）：
- 將模型當成**因果預測**
- 提升系統品質 → 必然提升使用 → 必然產生影響（因果）
- **問題**：忽略其他因素

**Seddon (1997) 的批判**：

Seddon 指出 D&M 模型混淆兩種模型：
1. **變異模型**（Variance Model）：X 變化導致 Y 變化（因果）
2. **過程模型**（Process Model）：描述事件發生順序（時序）

D&M 模型同時包含兩者，但未清楚區分

**實例說明**：

**時序關係**（過程）：
- 系統開發 → 使用者開始使用 → 產生影響
- 描述**發生順序**，不一定是因果

**因果關係**（變異）：
- 系統品質 ↑ → 滿意度 ↑
- X 的變化**導致** Y 的變化

**醫療 AI 實例**：

**誤用範例**：
- 假設：提升 AI 準確率（系統品質）→ 必然提升使用率
- 現實：準確率提升，但醫師仍不用（因為不信任 AI、工作流程不合）
- **錯誤**：忽略其他因素（信任、工作流程、組織文化）

**正確理解**：
- 系統品質是**必要但非充分**條件
- 還需要：訓練、支援、文化改變、激勵機制

**研究建議**：
- 區分過程描述 vs 因果預測
- 測試因果關係時需要嚴謹設計（實驗、縱貫）
- 加入調節變數（情境因素）

### 疑點 2：「使用」是成功指標還是前置因素？

**D&M 定位**：使用是**成功指標**之一
**批判觀點**：使用應該是**前置因素**，不是成功本身

**爭議**：

**D&M 觀點**：
- 使用是成功的表現（用得多 = 成功）
- 使用與滿意度並列為成功指標

**批判觀點**（Seddon等）：
- 使用只是**手段**，不是**目的**
- 真正的成功是**影響**（效益）
- 使用是達成影響的**前置條件**

**不同情境的問題**：

**自願使用情境**：
- 使用 = 成功指標（合理）
- 邏輯：願意用 = 認為有價值

**強制使用情境**：
- 使用 ≠ 成功指標（不合理）
- 邏輯：被迫用 ≠ 有價值
- **例外**：高使用但低滿意度（強制使用的 ERP）

**Seddon (1997) 建議**：

區分兩種模型：
1. **自願使用**：使用是成功指標（因為可選擇）
2. **強制使用**：使用是前置因素，影響才是成功

**修正模型**（強制使用情境）：

```
系統品質 ──→ 知覺有用性 ──→ 使用者滿意度 → 淨效益
資訊品質 ──→ 知覺易用性 ──┘                (Net Benefits)
```

移除「使用」，直接看「淨效益」

**醫療 AI 實例**：

**自願使用**（個人 AI App）：
- 使用率 = 成功指標（醫師覺得有用才用）

**強制使用**（醫院強制 AI）：
- 使用率 ≠ 成功指標（被迫用，可能陽奉陰違）
- 真正成功：診斷品質提升（淨效益）

**測量建議**：
- 自願使用：測量使用頻率、深度
- 強制使用：測量使用品質、效益（而非只看使用率）

### 疑點 3：時間動態被忽略 - 成功會隨時間改變

**D&M 假設**：靜態模型（某時間點的快照）
**實務現實**：IS 成功隨時間動態變化

**時間動態問題**：

**階段 1（蜜月期）**：
- 系統剛上線：新鮮感、期望高
- 使用率：高（好奇）
- 滿意度：可能高（蜜月效應）
- 影響：尚未顯現（需要時間）

**階段 2（幻滅期）**：
- 發現問題：bug、不符期望
- 使用率：下降（挫折）
- 滿意度：下降（失望）
- 影響：負面（干擾工作）

**階段 3（穩定期）**：
- 習慣養成、問題修復
- 使用率：穩定
- 滿意度：穩定（期望調整）
- 影響：正面顯現

**Gartner Hype Cycle 相似性**：

```
期望/滿意度
    ↑
    |    /\
    |   /  \___        ___---
    |  /      \    __/
    | /        \__/
    |/______________→ 時間
    技術  期望  幻滅  啟蒙  生產力
    觸發  膨脹  低谷  爬升  高原期
```

**測量時機影響結果**：

**T1（上線 3 個月）**：
- 使用率：80%（新鮮感）
- 滿意度：4.5/5（蜜月期）
- **結論**：成功！

**T2（上線 6 個月）**：
- 使用率：40%（幻滅）
- 滿意度：3.0/5（失望）
- **結論**：失敗！

**T3（上線 12 個月）**：
- 使用率：60%（穩定）
- 滿意度：4.0/5（期望調整）
- **結論**：中等成功

**問題**：不同時間評估，結論完全不同！

**研究建議**：
- **多時點測量**：T1, T2, T3... 追蹤變化
- **縱貫研究**：理解動態過程
- **脈絡說明**：報告測量時機（上線多久？）

**醫療 AI 實例**：

某醫院評估 AI 系統「成功」：
- 僅在上線 1 個月評估（蜜月期）→ 高滿意度
- **問題**：6 個月後使用率暴跌（忽略時間動態）

**建議評估時機**：
- 短期（3 個月）：系統品質、資訊品質
- 中期（6-12 個月）：使用、滿意度
- 長期（12+ 個月）：個人影響、組織影響

## 📊 研究方法

### 文獻回顧方法

**研究類型**：系統性文獻回顧（Systematic Literature Review）

**資料來源**：
- 7 個主要 IS 期刊（1981-1987）
- 180+ 篇 IS 成功實證研究

**分析步驟**：

**步驟 1**：識別成功測量指標
- 提取每篇研究使用的「成功」測量
- 分類：系統品質、資訊品質、使用等

**步驟 2**：歸納維度
- 將相似測量歸類
- 形成六大維度

**步驟 3**：建立因果關係
- 分析研究報告的變數關係
- 歸納因果路徑（品質 → 使用 → 影響）

**步驟 4**：整合模型
- 綜合所有路徑
- 提出整合性 IS 成功模型

### 六大維度的測量整理

**1. 系統品質（180 篇中有 100+ 篇測量）**

常見指標：
- 易用性（Ease of Use）
- 便利性（Convenience）
- 可靠性（Reliability）
- 回應時間（Response Time）
- 系統彈性（Flexibility）

**2. 資訊品質（100+ 篇測量）**

常見指標（Bailey & Pearson 影響）：
- 準確性（Accuracy）
- 時效性（Timeliness）
- 相關性（Relevance）
- 完整性（Completeness）
- 可理解性（Understandability）

**3. 使用（60+ 篇測量）**

測量方式：
- 客觀：系統使用時間、登入次數
- 主觀：自陳使用頻率
- 爭議：強制 vs 自願使用

**4. 使用者滿意度（70+ 篇測量）**

測量方式：
- 單一題項：整體滿意度
- 多題項：Bailey & Pearson 39 因素

**5. 個人影響（30+ 篇測量）**

常見指標：
- 決策品質（Decision Quality）
- 工作績效（Job Performance）
- 生產力（Productivity）
- 學習（Learning）

**6. 組織影響（20+ 篇測量）**

常見指標：
- 成本降低（Cost Reduction）
- 整體生產力（Overall Productivity）
- ROI（投資報酬率）
- 競爭優勢（Competitive Advantage）

## 🔍 理論影響

### 對 IS 研究的影響

| 年代 | 影響 | 代表研究 |
|------|------|---------|
| **1992-2000** | 成為 IS 評估標準 | 大量實證驗證 |
| **2003** | 模型更新 | DeLone & McLean (2003) 更新版 |
| **2000-2010** | 特定系統應用 | ERP、電商、知識管理系統 |
| **2010-現在** | 持續主導地位 | 30,000+ 引用，持續影響 |

### 引用趨勢

- **1992-2000**：確立經典地位（2,000+ 引用）
- **2000-2010**：快速成長（10,000+ 引用）
- **2010-現在**：持續主導（30,000+ 引用）

### 2003 更新版本

**DeLone & McLean (2003)** 根據 10 年研究更新模型：

**主要修正**：

**1. 加入「服務品質」**：
- 原版：系統品質、資訊品質
- 更新：+ 服務品質（Service Quality）
- 理由：IS 部門的服務越來越重要

**2. 合併「個人影響」和「組織影響」為「淨效益」**：
- 原版：分開測量
- 更新：合併為「淨效益」(Net Benefits)
- 理由：影響層次模糊，整合更簡潔

**3. 明確「使用」的測量**：
- 強調「使用意圖」（自願使用情境）
- 或「使用品質」（強制使用情境）

**更新後模型**：

```
資訊品質 ──┐
系統品質 ──┼──→ 使用/使用意圖 ──┐
服務品質 ──┘         ↕            ├──→ 淨效益
                 使用者滿意度 ────┘   (Net Benefits)
```

---

## ⚠️ 理論限制

1. **因果方向爭議**：雙向關係（使用 ↔ 滿意度）難以驗證
2. **情境限制**：自願 vs 強制使用適用性不同
3. **時間動態忽略**：靜態模型，未考慮隨時間變化
4. **測量挑戰**：「影響」難以客觀測量（尤其組織層次）
5. **過度簡化**：六維度無法涵蓋所有成功面向

## 🎯 實務啟示

### 對系統評估者

1. **多維度評估**
   - 不只看使用率或滿意度
   - 全面評估六大維度
   - 理解維度間關聯

2. **診斷問題根源**
   - 使用率低 → 檢查系統品質、資訊品質
   - 滿意度低 → 找出具體不滿因素
   - 無效益 → 檢查使用方式是否正確

3. **分階段評估**
   - 短期：品質
   - 中期：使用、滿意度
   - 長期：影響

### 對醫院 AI 推動者

1. **建立評估框架**
   - 系統品質：AI 可靠性、回應速度
   - 資訊品質：AI 準確率、相關性
   - 使用：醫師使用頻率、深度
   - 滿意度：醫師整體滿意度
   - 個人影響：診斷品質、時間節省
   - 組織影響：醫療品質、成本效益

2. **持續監控**
   - 定期測量六大維度
   - 追蹤變化趨勢
   - 早期發現問題

3. **整合改善**
   - 品質問題 → 技術改善
   - 使用問題 → 訓練、支援
   - 效益問題 → 工作流程優化

### 對研究者

1. **完整測量**
   - 涵蓋六大維度（或至少多個維度）
   - 避免單一指標

2. **因果檢驗**
   - 驗證 D&M 提出的因果路徑
   - 探討調節變數

3. **情境擴展**
   - 不同類型 IS（雲端、AI、行動）
   - 不同使用情境（自願、強制）

## 🔬 現代應用

### 雲端/SaaS 評估

**D&M 模型應用**：

- **系統品質**：可用性（uptime）、效能
- **資訊品質**：資料準確性、即時性
- **服務品質**：客戶支援、問題解決
- **使用**：登入頻率、功能使用率
- **滿意度**：NPS（淨推薦值）
- **淨效益**：生產力提升、成本節省

### 醫療 AI 成功評估

**完整評估框架**：

**系統品質**：
- AI 系統可靠性（當機次數）
- 回應時間（< 5 秒）
- 整合度（與 EHR 整合）

**資訊品質**：
- AI 診斷準確率（85%+）
- 建議相關性（醫師評分）
- 解釋清晰度（可理解性）

**服務品質**：
- 技術支援回應時間
- 訓練品質
- 問題解決率

**使用**：
- 使用率（% 醫師使用）
- 使用頻率（每週幾次）
- 使用深度（使用哪些功能）

**滿意度**：
- 整體滿意度（1-7 分）
- 推薦意願（NPS）

**淨效益**：
- 個人：診斷時間節省、準確率提升
- 組織：醫療品質、成本效益、病人滿意度

## 💡 研究啟示

### IS 成功的多維本質

**核心洞察**：
- IS 成功不是單一指標
- 需要平衡多個維度
- 維度間有因果關聯

**實務意涵**：
- 避免單一 KPI（如只看使用率）
- 建立平衡計分卡
- 理解改善哪個維度最有效

### 從輸出到影響

**演化軌跡**：
- 過去：關注系統輸出（功能、效能）
- D&M：關注使用與滿意度（使用者觀點）
- 現代：關注影響（價值創造）

**焦點轉移**：
- 不只是「系統做了什麼」
- 而是「系統創造什麼價值」
